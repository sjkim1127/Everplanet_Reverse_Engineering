## 치트 벡터 분석

### Speed Hack
- **가능성**: 제한적 (타이밍 함수 직접 접근 어려움)
- **대상 함수**: QueryPerformanceCounter (0x8f4194)
- **방법**: 함수 후킹 또는 시간 가속
- **탐지 위험**: 중간 (HShield 메모리 스캔)

### God Mode
- **가능성**: 가능 (데미지 계산 함수 후킹)
- **대상 함수**: 아직 발견되지 않음
- **방법**: 데미지 적용 함수 NOP 또는 체력 설정
- **탐지 위험**: 높음

### Teleport
- **가능성**: 가능 (좌표 업데이트 함수 후킹)
- **대상 함수**: 아직 발견되지 않음
- **방법**: 위치 검증 우회
- **탐지 위험**: 높음 (서버 검증 가능성)

### Item Duplication
- **가능성**: 가능 (아이템 추가 함수 악용)
- **대상 함수**: 아직 발견되지 않음
- **방법**: 트랜잭션 레이스 컨디션
- **탐지 위험**: 높음 (서버 검증)

## 결론 및 다음 단계

이 EverPlanet 클라이언트는 표준적인 한국 온라인 게임의 보안 수준을 가지고 있습니다. droopl 프로젝트의 글로벌 서버 에뮬레이터를 기반으로 한국 버전용 서버 에뮬레이터를 개발하는 것은 충분히 가능합니다.

### 주요 차이점
- droopl: 글로벌 버전 (Scala 구현)
- 한국 버전: KRver 플래그로 구분되는 추가 필드들

### 추천 개발 방향
1. droopl 코드를 포크하여 한국 버전 패킷 구조로 수정
2. Ghidra를 활용한 심층 패킷 구조 분석
3. Frida를 이용한 동적 패킷 캡처 및 분석
4. 서버 에뮬레이터 완성 후 클라이언트-서버 통신 테스트

### 분석 제한사항
- 코드 obfuscation으로 인한 분석 어려움
- 동적 분석 미진행 (Frida 활용 필요)
- 암호화 분석 미완료

이 분석은 초기 정찰 단계이며, 심층 분석을 위해서는 더 많은 시간과 정적/동적 분석 도구의 활용이 필요합니다.
